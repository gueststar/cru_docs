NAME

cru_bpred - binary predicate type used in the cru library

SYNOPSIS

#include <cru/cru.h>

typedef int (*cru_bpred)(void *, void *, int *)

DESCRIPTION

A user-defined binary predicate takes a pair of operands and a pointer to an initially zero error code as arguments. The operands can be scalar values, pointers to heap allocated storage, or a combination. The operator must treat the operands as read only but may assign the error code.

Binary predicates model mathematical relations and are used as decision procedures in various contexts. See the NOTES for details.

RETURN VALUE

Viewed as a relation, a binary predicate returns zero if its operands are unrelated and a non-zero value if they are related.

ERRORS

If a binary predicate sets the error code to any non-zero value, the cru API function invoking it terminates early without completing, and propagates the same error code to its caller.

Applications can report POSIX error codes or define special purpose codes, but should not redefine any that clash with those reserved by cru as declared in the error_codes.h header file.

NOTES

Applications pass binary predicates to cru as pointers through

  • the bpred field in a cru_cbop (conditional binary operator) structure
  • the fissile field in a cru_splitter top level structure
  • and the equal field in a cru_order, which can belong to a cru_order_pair, a cru_classifier, or a cru_filter.

A conditional binary operator can be the postponement field in a cru_postponer or the e_prod field in a cru_crosser.

  • In a cru_postponer the predicate selects postponable edges for postponement by returning a non-zero value given and postponable edge label as its left operand the sibling as its right, but zero otherwise.
  • In a cru_crosser the predicate takes the labels of two edges each from a vertex in one of the graphs to be crossed, and returns non-zero only if a combined edge corresponding to them should be created in the resulting graph.
